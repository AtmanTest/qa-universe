## 📋 CONTEXTE TECHNIQUE
- Fichier principal : `index.html` (~11 500 lignes, ~490 KB), tout CSS/JS inline
- Pipeline news : `fetch-news.mjs` — collecte RSS/Atom, filtre QA, injecte bloc `const NEWS` dans index.html
- Tests régression : `tests-bugs.mjs` — suite Playwright, commande `node tests-bugs.mjs`
- Automatisation : `.github/workflows/update-news.yml` (cron UTC + workflow_dispatch)
- Backups : `index.backup.html`, `index.v2.backup.html`, `index.v3.3.backup.html`, `qa-universe-v1-complete.html`
- L'utilisateur est Thasin Jahangir, Consultant QA Senior, communique en français

## 🔴 PROBLÈMES PRIORITAIRES (ordre d'exécution)

### PRIORITÉ 1 — AUTOMATISATION NEWS
1a. Stabiliser `fetch-news.mjs` : vérifier les flux RSS/Atom, le filtrage QA, la déduplication et le rejet explicite des sujets hors périmètre (crypto, travel, sport, finance grand public, lifestyle).
1b. Créer ou corriger `.github/workflows/update-news.yml` avec :
    - cron UTC (ex: `0 6 * * *`)
    - `workflow_dispatch` pour déclenchement manuel
    - logs lisibles par étape
    - commit contrôlé (git add / commit / push) uniquement si des articles nouveaux existent
    - échec explicite (`exit 1`) si aucun article exploitable n'est trouvé
1c. Objectif à terme : sortir les données news de `index.html` vers `data/news.json` ou `data/news.mjs`, pour réduire le risque de corruption du fichier principal.

### PRIORITÉ 2 — SECTION DAILY NEWS
2a. Créer la section `Daily News` en haut de l'onglet News, AVANT le flux principal.
2b. Règles de sélection : 3 à 5 contenus majeurs du jour, les plus récents et/ou les plus stratégiques pour le QA/software testing (une par catégorie si possible : AI Testing, QA News, Security Risk, Training, Thought Leadership).
2c. Rendu distinct : titre `Daily News`, style visuel différencié (badge, bordure, card mise en avant), séparation visuelle nette avec le flux principal.
2d. Le flux principal sous `Daily News` doit rester large, sans limitation artificielle, couvrant tout l'écosystème QA/software testing.
2e. `Daily News` doit fonctionner en FR et EN (attributs `data-i18n` requis).

### PRIORITÉ 3 — RESPONSIVE MOBILE
3a. Burger menu : s'ouvre, affiche les 6 onglets, se referme correctement (tap outside / tap X / tap item).
3b. Contenu onglets : aucun contenu tronqué ou qui dépasse sur mobile (Dashboard, Tools, ISTQB Prep, Labs, Training, News).
3c. Chat IA : fenêtre `position:fixed, right:88px` trop large — adapter pour mobile (max-width: 100vw, right: 0 ou 8px, scrollable).
3d. Avatar TJ dans la navbar : masqué sur mobile (`display:none` en dessous du breakpoint mobile).
3e. Navbar overflow : `.nav-actions` ne doit pas déborder sur mobile.
3f. Aucun scroll horizontal sur aucune vue (desktop ou mobile), aucun panel qui déborde.

### PRIORITÉ 4 — MODE LIGHT COMPLET
4a. Tous les composants doivent être correctement stylés sous `[data-theme="light"]` : fond blanc/clair, texte foncé, bordures adaptées.
4b. Composants à vérifier explicitement : cartes news, Daily News cards, filtres, badges, dropdowns, assistant IA, ticker, boutons, panels, états vides.
4c. Aucune régression de contraste, lisibilité, hover/focus ou overflow en mode light.

### PRIORITÉ 5 — ASSISTANT IA 🤖
5a. Animation de vibration toutes les 10s : s'assurer que l'état CSS n'est pas écrasé par d'autres classes d'état.
5b. Questions prédéfinies : défilement horizontal sur mobile (overflow-x: auto, scroll snap).
5c. Design bulles de chat : messages user/assistant différenciés (bulles, couleurs, alignement).
5d. Tests : ouverture, fermeture (tap outside, bouton X), saisie, envoi, réponse, scroll interne du panel, coexistence avec thème/langue.

### PRIORITÉ 6 — I18N ET THÈME TRANSVERSES
6a. Vérifier que tous les éléments `data-i18n` s'affichent correctement après changement de langue (FR → EN et EN → FR).
6b. Vérifier que la bascule dark/light fonctionne sur TOUS les composants : footer, assistant, drawer, tools cards, hero card, Daily News, ticker.
6c. Aucun contenu mixte FR/EN visible après rerender, notamment dans les sections news, outils, labs, assistant, labels dynamiques, états UI.
